# 前提
* セットメニューは必ず単品メニューとして存在するものの組み合わせで構成されている。

# テーブル説明

## orders
注文情報を管理するテーブル。

注文番号、合計金額、預かり金、注文日時を持つ。会計レジは `checkouts_id`、支払い方法は `payments_id` で管理する。

## payments
支払い方法を管理するテーブル。

現金、クレジットカード、電子マネーなどの支払い方法名を `name` に保存する。

## order_menus
注文とメニューの明細を管理する中間テーブル。

1つの注文に複数のメニューを紐づけるために使用する。注文は `orders_id`、メニューは `menu_id` で参照し、注文数量を `quantity` に保存する。

## menus
注文可能なメニュー全体を管理するテーブル。

単品メニューとセットメニューの両方をこのテーブルで扱う。メニュー名、単価、税込価格を管理する。

## menu_breakdowns
セットメニューの内訳を管理するテーブル。

セットメニューがどの単品メニューで構成されているかを表す。`set_id` と `single_menu_id` はどちらも `menus` を参照し、セット内に含まれる単品メニューの数量を `quantity` に保存する。

## checkouts
会計レジを管理するテーブル。

レジの番号を管理する。お会計時に利用されたレジ番号を `name` に保存する。

# 確認したい点
* ordersテーブルにお釣りの情報を追加した方が良いか。今回は、合計金額と預り金から特定できるため不要と判断した。
* セットメニューと単品メニューの管理の仕方について、今回は、menu を自己参照する形を選んだが、もっと良い方法や選択肢がないか。
